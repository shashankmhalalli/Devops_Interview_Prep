# Project 5 — Terraform + Ansible on AWS

Provision an AWS VPC + EC2 instances with Terraform, then configure Nginx on
them automatically using Ansible.

---

## Architecture

```
Internet
    │
    ▼
Internet Gateway
    │
    ▼
VPC (10.0.0.0/16)
    ├── Public Subnet 1 (10.0.1.0/24) — ap-south-1a
    │       └── EC2 web-1  ←── Security Group: SSH/HTTP/HTTPS
    └── Public Subnet 2 (10.0.2.0/24) — ap-south-1b
            └── EC2 web-2  ←── Security Group: SSH/HTTP/HTTPS
```

Ansible connects to each EC2 instance via SSH and:
- Updates the system packages
- Installs Nginx
- Deploys a custom `index.html`
- Verifies Nginx returns HTTP 200

| Tool        | Version    |
|-------------|------------|
| Terraform   | >= 1.3     |
| Ansible     | >= 2.14    |
| AWS CLI     | configured |
| jq          | any        |

```bash
# Verify
terraform version
ansible --version
aws sts get-caller-identity
jq --version
```

Also ensure you have an SSH key pair:
```bash
ls ~/.ssh/id_rsa ~/.ssh/id_rsa.pub   # must exist
# Generate if missing:
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa
```

## Step-by-step Workflow

### 1 — Terraform: init

```bash
cd terraform/
terraform init
```

### 2 — Terraform: plan (dry-run)

```bash
terraform plan
```

Review the plan. You should see ~10 resources being created.

### 3 — Terraform: apply

```bash
terraform apply
# Type 'yes' when prompted
```

On success, you'll see outputs including `instance_public_ips`.

### 4 — Generate Ansible Inventory

```bash
cd ..
chmod +x scripts/generate_inventory.sh
./scripts/generate_inventory.sh
```

This reads `terraform output -json instance_public_ips` and writes
`ansible/inventory.ini` automatically.

Alternatively, copy the `ansible_inventory_block` output manually.

### 5 — Verify SSH Access

```bash
cd ansible/
ansible web_servers -m ping
```

Expected output:
```
1.2.3.4 | SUCCESS => { "ping": "pong" }
5.6.7.8 | SUCCESS => { "ping": "pong" }
```

### 6 — Run the Ansible Playbook

```bash
ansible-playbook -i inventory.ini playbook.yml
```

The playbook will:
1. Wait for SSH to be ready
2. Update apt packages
3. Install Nginx
4. Write a custom server block config
5. Copy `files/index.html` to `/var/www/html/`
6. Start and enable Nginx
7. Smoke-test: verify `http://localhost` returns 200

### 7 — Verify in Browser

Open each public IP from Terraform output:
```
http://<EC2_PUBLIC_IP_1>/
http://<EC2_PUBLIC_IP_2>/
```

You should see the deployed webpage.

## File Structure

```
project-5-terraform-ansible/
├── terraform/
│   ├── main.tf            # VPC, subnets, SGs, EC2
│   ├── variables.tf       # All input variables with defaults
│   ├── outputs.tf         # Public IPs, inventory block
│   └── terraform.tfvars   # Your overrides
├── ansible/
│   ├── ansible.cfg        # SSH settings, defaults
│   ├── inventory.ini      # Auto-generated by script
│   ├── playbook.yml       # Main playbook
│   ├── templates/
│   │   └── nginx_site.conf.j2  # Nginx server block
│   └── files/
│       └── index.html     # Static website to deploy
└── scripts/
    └── generate_inventory.sh   # Terraform → Ansible bridge
```
